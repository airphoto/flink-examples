1:p1_example
    本例中事件模板为当五秒中出现三次名字为login的数据之后，接下来再出现两次名字为login的数据，则触发该事件
    数据源格式 ： "1,login,"+System.currentTimeMillis();

2:p2_doors 人员离场超时预警
    从员工刷卡出门开始，与当前时间比较，相差90分钟异常，则监测次员工的刷卡进入状态，如果没有监测到则进行离场预警
    员工90分钟内出入3次，报警频繁出入
    此案例是可以检测到如果有out的数据，但是90分钟后没有in的数据就会自动报警的，是由outputTag输出的
    数据源格式:
       id,doorid,status,eventType,userid,datetime
       1,1,out,1,1,1635332817000
       1,1,in,1,1,1635332827000
       1,1,out,1,2,1635332837000
       1,1,out,1,3,1635332837000

       1,1,out,1,4,1635332817000
       1,1,in,1,4,1635332827000
       1,1,out,1,4,1635332817000
       1,1,in,1,4,1635332827000
       1,1,out,1,4,1635332817000
       1,1,in,1,4,1635332827000

       其中userid=1模拟的是正常出入的情况
       userid=2和userid=3是异常出入的情况
       userid=4描述的是频繁出入的秦光
